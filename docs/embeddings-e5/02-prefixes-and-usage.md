# Магия префиксов и использование E5

> [!CAUTION]
> Использование E5 v2 без префиксов — самая частая причина плохого качества поиска. 

## 1. Зачем нужны префиксы?
E5 была обучена различать ассиметричные задачи. Вопрос и ответ — это разные сущности. Префиксы помогают модели понять, в какой "роли" выступает текущий текст.

## 2. Канонические префиксы

### `query: ` (Для запросов)
Используй его, когда превращаешь в вектор вопрос пользователя или поисковую фразу.
*Пример:* `query: Как приготовить шарлотку?`

### `passage: ` (Для документов)
Используй его, когда индексируешь базу знаний, описания героев или логи событий.
*Пример:* `passage: Шарлотка — это яблочный пирог, требующий 3 яйца и стакан муки.`

## 3. Влияние на точность
Исследования показывают:
- **С правильными префиксами**: Точность (MRR@10) ~0.55
- **Без префиксов**: Точность падает до ~0.42
- **С перепутанными префиксами**: Поиск превращается в случайный выбор.

## 4. Практические советы по внедрению

### В коде (Python)
Всегда делай обертку над методом `encode`:

```python
def get_emb(text, is_query=False):
    prefix = "query: " if is_query else "passage: "
    return model.encode(prefix + text, normalize_embeddings=True)
```

### В базе данных (Qdrant)
В Qdrant мы храним только `passage:` векторы. Префикс `query:` добавляется только "на лету" в момент поиска.

## 5. Длина и контекст
Префиксы съедают часть контекстного окна (несколько токенов). Учитывай это при нарезке чанков. Если твой чанк ровно 512 токенов, префикс вытолкнет последние слова за пределы окна.

## 6. Что если я хочу найти похожие документы? (Doc-to-Doc)
Если ты хочешь найти документ, похожий на текущий (а не ответ на вопрос), используй префикс `passage: ` для обеих сторон. Это превратит задачу в поиск дубликатов или близких тем.
