# Оптимизация E5: Скорость и Память

Модель `multilingual-e5-large` весит 1.2 GB и требует значительных ресурсов при массовой индексации. Мы разберем, как заставить её летать.

## 1. Использование GPU и Metal (MPS)
На Mac с чипами M1/M2/M3/M4 использование ядра Apple Silicon ускоряет инференс в 5-10 раз.

```python
# Автоматический выбор устройства
import torch
device = "mps" if torch.backends.mps.is_available() else "cpu"
model = SentenceTransformer('intfloat/multilingual-e5-large', device=device)
```

## 2. Батчинг (Batching)
Никогда не закодируй документы по одному. Отправляй их пачками.
- Оптимальный размер: `batch_size=32` или `64`.
- Это позволяет модели обрабатывать несколько текстов параллельно внутри одного прохода нейросети.

## 3. Квантование (Quantization)
Если тебе не хватает RAM, можно использовать квантованные версии модели.

### FP16 (Половинная точность)
Стандарт для большинства задач. Экономит 50% памяти почти без потери точности.
```python
model.encode(texts, convert_to_numpy=True, output_value="sentence_embedding")
# Torch по умолчанию использует fp32, если не указать иное при загрузке модели
```

### INT8 (Целочисленное квантование)
Сжимает модель в 4 раза. Хорошо для CPU, но может быть сложнее настроить для MPS.

## 4. Нормализация (Normalization)
Всегда выставляй `normalize_embeddings=True`.
- **Зачем?** Это превращает длину вектора в 1.
- **Профит**: В базе данных (Qdrant) вместо сложного `Euclidean Distance` можно использовать молниеносный `Dot Product`.

## 5. Оптимизация Qdrant (On-disk)
Если векторов миллионы, они не влезут в RAM.
- Используй `mmap: true` в настройках коллекции Qdrant.
- Это позволит ОС подгружать векторы с диска только в момент обращения.

## 6. Кэширование
Если ты часто перезапускаешь пайплайн с тем же героем:
- Считай эмбеддинг описания героя один раз.
- Сохрани его в Neo4j или Redis как blob.
- Не вызывай LLM повторно для того, что не менялось.
