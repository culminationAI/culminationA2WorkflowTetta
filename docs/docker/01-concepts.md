# Docker: Основные концепции

Чтобы использовать Docker "на полную мощь", нужно понимать, как он устроен внутри.

## 1. Образ (Image) vs Контейнер (Container)
- **Образ** — это чертеж или слепок системы (как `.iso` или `class` в ООП). Он неизменяем.
- **Контейнер** — это живой экземпляр образа (как запущенная программа или `object`).

## 2. Dockerfile: Твой рецепт системы
Dockerfile — это текстовый файл с инструкциями по сборке образа.

**Пример для нашего Python-приложения:**
```dockerfile
# 1. Берем легкую базу
FROM python:3.11-slim

# 2. Указываем рабочую папку
WORKDIR /app

# 3. Копируем зависимости
COPY requirements.txt .

# 4. Устанавливаем (без кэша, чтобы образ был меньше)
RUN pip install --no-cache-dir -r requirements.txt

# 5. Копируем код
COPY . .

# 6. Команда запуска
CMD ["python", "main.py"]
```

## 3. Слои (Layers) и Кэширование
Каждая команда в Dockerfile создает **слой**. 
- Docker кэширует слои. 
- Если `requirements.txt` не менялся, шаг установки `pip` будет пропущен при следующей сборке.
- **Правило**: Самые редко меняющиеся команды (установка системных пакетов) ставь в начало.

## 4. Громкости (Volumes)
Контейнеры смертны. Если удалить контейнер, данные внутри него пропадут.
- **Volumes** выносят данные (папки с БД Neo4j/Qdrant) на твой диск.
- Данные живут вечно, даже если ты пересоздаешь контейнеры.
