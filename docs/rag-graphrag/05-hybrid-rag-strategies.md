# Hybrid RAG Strategies: Векторы + Графы

Гибридный RAG — это архитектурная вершина. Она объединяет интуитивное понимание векторного поиска и логическую строгость графов.

## 1. Почему Hybrid?
- **Vector (Qdrant)**: Хорош для вопросов "О чем этот текст?", когда нет четких имен.
- **Graph (Neo4j)**: Хорош для вопросов "Как связаны А и Б через три звена?".
- **Hybrid**: Идеален для глубокого анализа. "Найди всех врагов Ивана и проанализируй их общие мотивы".

## 2. Паттерны объединения

### A. Two-Path Retrieval (Параллельный поиск)
1.  Запускаем векторный поиск (Top-5 кусков).
2.  Запускаем графовый поиск (Sub-graph из 2-х прыжков от ключевых сущностей вопроса).
3.  **Synthesis**: LLM получает оба набора данных и формирует ответ.

### B. Graph-First Retrieval
1.  Сначала ищем сущность в графе.
2.  Достаем все факты о ней.
3.  Используем эти факты, чтобы сделать векторный поиск в Qdrant более точным (например, через фильтрацию по Entity ID).

### C. Vector-First Entry
1.  Векторный поиск находит "входную точку" (например, описание события).
2.  Система извлекает ID сущностей из этого куска.
3.  Переходит в Neo4j, чтобы «расширить» контекст соседними связями.

## 3. Объединение результатов (Fusion)
Как смешать результаты из разных баз?
- **Reciprocal Rank Fusion (RRF)**: Алгоритм, который дает больший вес документам, попавшим в топ в обоих видах поиска.
- **LLM Reranking**: Самый точный (но дорогой) способ. Подаем 20 результатов LLM и просим выбрать 5 лучших.

## 4. Кейс: Temporal Hybrid RAG
В нашем проекте «Интерпретация» мы добавили измерение времени.
- Векторы хранят «дух» воспоминаний.
- Граф хранит «таймлайн» фактов.
- Система выбирает: "Мне нужно знать ЧТО произошло (Neo4j) или КАК это ощущалось (Qdrant)".
