# RAG Evaluation: Как измерить успех?

Создать RAG — просто. Создать качественный RAG — сложно. Современный стандарт 2026 года требует использования фреймворков типа **RAGAS** для объективной оценки.

## 1. "RAG Triad" (Триада оценки)
Оценка строится вокруг трех ключевых звеньев:

### A. Context Precision (Точность контекста)
- **Вопрос**: Насколько извлеченные документы релевантны вопросу?
- **Цель**: Минимизировать шум. Агент не должен получать 50 страниц текста, если ответ на 1-й.

### B. Faithfulness (Верность)
- **Вопрос**: Насколько ответ соответствует извлеченному контексту?
- **Цель**: Борьба с галлюцинациями. Агент должен отвечать "Я не знаю", если в контексте нет информации.

### C. Answer Relevance (Релевантность ответа)
- **Вопрос**: Насколько ответ соответствует вопросу пользователя?
- **Цель**: Полезность. Даже если ответ «верный» по базе, он может не отвечать на суть вопроса.

## 2. Метрики RAGAS
В 2026 году мы используем следующие метрики:
1.  **Context Recall**: Все ли необходимые факты попали в контекст?
2.  **Context Precision**: Какой % извлеченного текста был полезен?
3.  **Faithfulness**: Проверка фактов из ответа по источнику (LLM-as-a-Judge).
4.  **Answer Semantic Similarity**: Насколько близок ответ к эталонному (если есть Golden Dataset).

## 3. Золотой датасет (Golden Dataset)
Для "Ultimate" системы необходимо создать набор пар "Вопрос-Ответ-Контекст", проверенный человеком. Это позволяет:
- Замерять регрессию при обновлении модели.
- Сравнивать разные способы эмбеддинга (E5 vs BGE).
- Оптимизировать параметры `top_k`.

## 4. Непрерывная оценка
В продакшене мы используем **Observability** инструменты (LangSmith, DeepEval):
- Мы логируем каждый "Retrieval Score".
- Если оценка Faithfulness падает ниже 0.8, система алертит разработчиков.
