# JSON Schema Mastery: Промпт внутри структуры

JSON Schema — это не просто техническое описание. Для ИИ это **карта реальности**. Чем лучше вы опишете схему, тем "умнее" будет ответ модели.

## 1. Основы JSON Schema
Схема говорит модели: "Тут должен быть объект с такими полями".
Пример простейшей схемы:
```json
{
  "type": "object",
  "properties": {
    "name": { "type": "string" },
    "age": { "type": "integer" }
  },
  "required": ["name", "age"],
  "additionalProperties": false
}
```

## 2. Сила `description`
В 2026 году `description` внутри схемы — это самый важный промпт.
```json
"sentiment": {
  "type": "integer",
  "description": "Оценка настроения текста от 1 (очень грустно) до 10 (восторг)."
}
```
ИИ читает эти описания и использует их как инструкции. Это гораздо эффективнее, чем писать инструкции в общем системном промпте.

## 3. Ограничения (Enums и Ranges)
Используйте `enum`, чтобы загнать модель в жесткие рамки:
```json
"status": {
  "type": "string",
  "enum": ["pending", "active", "deleted"]
}
```
Теперь модель не сможет выдать "waiting" или "in_progress".

## 4. Рекурсивные структуры
ИИ отлично справляется с вложенностью. Вы можете описать "Дерево категорий", где у категории есть свойство `subcategories`, которое является массивом объектов того же типа. 
*Примечание*: Современные Structured Outputs (OpenAI) поддерживают рекурсию ограниченной глубины.

## 5. Трюк: `additionalProperties: false`
Всегда устанавливайте этот параметр в `false`. Это заставляет модель строго следовать схеме и не добавлять отсебятину типа `"comment": "Я закончил выполнение задачи"`.
